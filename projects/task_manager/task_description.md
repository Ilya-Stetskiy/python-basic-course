Personal Task Manager
=====================

## Описание проекта
Напишите программу на языке Python, реализующую персональный менеджер задач. Эта программа должна обладать возможностями управления своими задачами, установки приоритетов и хранения сведений о задачах. Менеджер задач должен обеспечивать базовую функциональность добавления, перечисления, обновления и удаления задач.  
Этот проект направлен на оценку уровня владения языком Python, включая обработку файлов, структур данных и базовое взаимодействие с пользователем.

## Требования к проекту

### 1. Данные  
Каждая задача должна иметь следующие атрибуты:
   - Заголовок (строка)
   - Описание (строка)
   - Срок выполнения (дата или строка)
   - Приоритет (например, низкий, средний, высокий)

### 2. Функциональность
   - **Добавление задач:** разрешить пользователям добавлять задачи со всеми необходимыми атрибутами.
   - **Отображение списока задач:** отображение списка задач с их подробной информацией (название, описание, срок выполнения, приоритет).
   - **Обновление задач:** разрешить пользователям обновлять атрибуты задачи (название, описание, срок выполнения, приоритет).
   - **Удаление задач:** разрешить пользователям удалять задачу по ее названию и/или номеру.

### 3. Хранение данных
Хранить данные необходимо в структурированном формате (например, JSON, текстовый файл, база данных, др.) для сохранения между запусками программы. 

### 4. Пользовательский интерфейс
   - Реализуйте базовый пользовательский интерфейс командной строки (CLI) для взаимодействия с диспетчером задач.
   - *(опционально^2)* Релизуйте графический интерфейс для взаимодействия с диспечером задач.
   - *(опционально^2)* Реализуйте web-сервис с интерфейсом для взаимодействия с диспечером задач.

### 5. Расширенные функции *(опционально)*
Необходимо внедрить расширенные функции для улучшения работы диспетчера задач, такие как:
   - Категоризация задач по проектам или меткам.
   - Параметры приоритизации задач и сортировки.
   - Напоминания или уведомления о сроках выполнения.
   - Функции экспорта и импорта данных.
   - *(опционально)* Любые дополнительные функции, которые делают диспетчер задач более универсальным и удобным в использовании.

### 6. Обработка ошибок
Внедрить обработку ошибок для решения распространенных проблем, таких как недопустимый ввод, ошибки во время обработки файлов и другие.

## Критерии оценки
Ваш проект будет оцениваться на основе следующих критериев:
- Правильность и полнота программы.
- Правильная организация кода, включая функции и управление данными.
- Эффективное использование структур данных для хранения задач и манипулирования ими.
- Удобный интерфейс.
- Правильно реализованные расширенные функции (при наличии).
- Правильно реализованная обработка ошибок.

## Сдача работы
Вы должны предоставить свой скрипт на Python (файл .py) (или несколько скриптов) вместе с любыми дополнительными файлами, необходимыми для хранения данных, и документом README.MD, который объясняет, как запустить программу, взаимодействовать с ней, и любыми дополнительными инструкциями или информацией.
